# VERTEX STUDIO — UX ROADMAP & CLAUDE CODE PLAYBOOK

## CONTEXTO DEL PROYECTO

Vertex Studio es una plataforma de virtual influencers con AI image editing.
Stack: React 18 + TypeScript + Tailwind CSS + Vite.
Design system: Plasma Shift palette, Playfair Display / DM Sans, glassmorphism.
Este roadmap prioriza cambios UX por impacto en activación de usuarios nuevos.

---

## PRIORIDAD DE EJECUCIÓN

Ejecutar en este orden estricto. Cada fase es un bloque independiente.
NO avanzar a la siguiente fase sin completar y testear la anterior.

## FASE 2 — REDUCIR FRICCIÓN EN FLUJOS CORE (Sesión 6-8)
> Cada decisión que le quitas al usuario nuevo es conversión que ganas.

### 2.1 Create Character — Defaults inteligentes
- Al entrar a Create Character, preseleccionar:
  - Render Style: "Photorealistic" (el más popular y el que da mejores resultados)
  - Gender: ninguno preseleccionado (decisión personal, no asumir)
  - Age: ninguno preseleccionado
- Marcar campos obligatorios con asterisco rojo: Name*, Render Style*, Gender*, Age*
- El botón "Next →" debe estar deshabilitado hasta que los obligatorios estén completos, con microcopy indicando qué falta.

### 2.2 Create Character — Preview dinámica
- El panel de Preview (derecha, 320px) debe actualizarse con cada selección:
  - Selecciono "Photorealistic" → Preview muestra ejemplo de ese estilo
  - Selecciono "Anime" → Preview cambia a ejemplo anime
  - Escribo nombre "Luna" → Preview muestra "Luna" en el header
- Eliminar el texto "Configure in Step 1" cuando ya ESTAMOS en Step 1.
- Cada render style necesita 1 imagen de ejemplo (puede ser estática, placeholder art).

### 2.3 Director — Ocultar complejidad
- Crear dos modos: "Simple" (default) y "Advanced" (opt-in).
- Modo Simple muestra:
  - Character selector
  - Pose (campo de texto o presets básicos)
  - Scene/Scenario
  - Botón Generate
- Modo Advanced agrega:
  - Camera presets (Portrait 85mm, Wide 24mm, etc.)
  - Lighting presets (Natural, Studio, Golden, Neon, etc.)
  - Face References
  - Characteristics (prompt técnico completo)
- Toggle en el header: `[Simple] [Advanced]` — el toggle "Auto" actual puede transformarse en esto.

### 2.4 AI Editor — Reducir herramientas visibles
- Mostrar las 5 herramientas principales en la barra lateral:
  - Free AI, Relight, Face Swap, Background, Enhance
- Las otras 5 (360°, Try-On, Scene, Style, Inpaint) van en un botón "More Tools +" al final de la barra.
- Al hacer clic en "More Tools +", se expande mostrando las 5 adicionales.
- Guardar preferencia del usuario en localStorage para que si expande, quede expandido.

### 2.5 Photo Session — Organizar vibes
- Agrupar los 14 vibes en categorías con headers:
  - **Social**: Selfies, GRWM, Stories, Foto Dump
  - **Fashion**: Editorial, Portrait, Street Style
  - **Lifestyle**: Creator, Lifestyle, Cozy Home
  - **Night/Outdoor**: Night Out, Date Night, Pool/Summer, Fitness
- Agregar numbered steps en el flujo:
  - Step 1: "Choose a base photo" (source selector)
  - Step 2: "Pick a vibe" (vibe grid)
  - Step 3: "Shoot" (generate button)
- Los 8 slots numerados en el bottom: agregar label "Generated shots will appear here"

## FASE 3 — NAVEGACIÓN & PIPELINE VISUAL (Sesión 9-10)
> Hacer que el pipeline sea obvio sin documentación.

### 3.1 Sidebar — Numerar el pipeline
En la sección PIPELINE del sidebar, agregar números:

```
PIPELINE
  ① Create Character
  ② Director
  ③ AI Editor
  ④ Photo Session

MANAGE
  Dashboard
  Gallery
  Characters
```

- Los números pueden ser badges circulares con el color accent (purple/pink del theme).
- Opcionalmente, conectar con línea vertical fina entre los items del pipeline.

### 3.2 Sidebar — Indicador de progreso
- Si el usuario no tiene personajes, mostrar badge en "① Create Character": "Start here"
- Si tiene personajes pero 0 hero shots, mostrar badge en "② Director": "Next step"
- Esto actúa como breadcrumb guiado sin necesidad de tutorial formal.

### 3.3 Dashboard — Quick Actions con orden
- Las 4 Quick Actions deben reflejar el orden del pipeline:
  - Card 1: "① Create Character" — "Design your virtual influencer"
  - Card 2: "② Direct Scene" — "Create a hero shot"
  - Card 3: "③ AI Editor" — "Edit and enhance images"
  - Card 4: "④ Photo Session" — "Generate themed photo sets"
- Si el usuario no ha completado Step 1, las cards 2-4 muestran un estado "Complete Step 1 first" con opacidad reducida (no deshabilitadas, pero visualmente secundarias).

### 3.4 Verificar contraste WCAG
- Auditar todos los textos secondary (lavanda rgb 168,152,184) contra fondo oscuro (rgb 8,7,12).
- Herramienta: usar extensión de browser o script con contrast ratio checker.
- Target: AA compliance mínimo (ratio 4.5:1 para texto normal, 3:1 para texto grande).
- Si no pasa, ajustar los secondary colors subiendo luminosidad.

## FASE 4 — POLISH & GALLERY FIX (Sesión 11-12)
> Calidad de vida y data integrity.

### 4.1 Gallery — Thumbnails y metadata
- Cada item de galería debe mostrar:
  - Thumbnail de la imagen (si existe en el storage)
  - Nombre del personaje usado (si fue vinculado)
  - Tipo de edición (Relight, Face Swap, etc.)
  - Fecha legible ("Mar 14" → "March 14, 2026")
- Implementar preview al hover (imagen más grande en tooltip o modal ligero).

### 4.2 Gallery — Naming automático
- Al guardar una creación, generar nombre automático:
  - Formato: "{Character} · {Tool} · {Key Setting}"
  - Ejemplo: "Luna · Relight · Golden Hour"
  - Ejemplo: "No Character · Free AI · Cinematic"
- Permitir rename manual con doble clic.

### 4.3 Gallery — Fix de tagging
- Investigar el pipeline de guardado: cuando se genera/edita una imagen, ¿se guarda el metadata de qué herramienta se usó y qué personaje?
- Si no se guarda, agregar campos al modelo:
  - `toolUsed: string` (relight, faceswap, freeedit, etc.)
  - `characterId: string | null`
  - `editSettings: Record<string, any>` (configuración usada)
- Backfill los 132 items existentes si es posible (basándose en logs o timestamps).

### 4.4 Delete confirmation
- Toda acción de borrado en Gallery y Characters requiere modal de confirmación:
  ```
  "Delete this creation?"
  "This action cannot be undone."
  [Cancel] [Delete]
  ```
- El botón Delete debe ser rojo para señalar peligro.

### 4.5 Photo Session — Preview por vibe
- Al hacer hover sobre cada vibe, mostrar tooltip con 1-2 imágenes de ejemplo del estilo.
- Esto ayuda enormemente a la conversión: el usuario entiende qué va a obtener.
- Las imágenes de ejemplo pueden ser estáticas (generadas previamente para demo).

## NOTAS PARA CLAUDE CODE

### Cómo usar este archivo
1. Copia este archivo a tu directorio de Vertex Studio como `UX_ROADMAP.md`
2. En cada sesión de Claude Code, referencia la fase en la que estás:
   ```
   "Estoy trabajando en la Fase 0 del UX Roadmap. Ejecuta el punto 0.1: Unificar branding."
   ```
3. Claude Code debe:
   - Leer este archivo al inicio de cada sesión
   - Ejecutar los cambios punto por punto
   - Hacer commit por cada sub-punto (0.1, 0.2, etc.)
   - Testear que no rompe nada antes de avanzar

### Convenciones
- Commits: `fix(ux): [fase].[punto] — descripción corta`
  - Ejemplo: `fix(ux): 0.1 — unify branding to Vertex Studio`
  - Ejemplo: `feat(ux): 1.1 — add dashboard empty state with onboarding`
- Branch: `ux/roadmap-fase-N` (una branch por fase)
- Todo componente nuevo va en `src/components/ux/` para separar de features existentes

### Dependencias de datos
- Fase 0 y 1: solo UI, no requieren cambios de backend/API
- Fase 2: puede requerir imágenes placeholder para previews (guardar en `public/examples/`)
- Fase 3: solo UI
- Fase 4: requiere cambios en el modelo de datos (agregar campos de metadata)

### Criterio de "terminado" por fase
- Fase 0: Zero jerga interna visible, créditos visibles everywhere, tooltips en todos los iconos
- Fase 1: Un usuario con 0 data ve onboarding guiado en todas las pantallas
- Fase 2: Un usuario nuevo puede crear su primer personaje y hero shot en <5 minutos sin confusión
- Fase 3: El pipeline 1→2→3→4 es visualmente obvio sin explicación
- Fase 4: La galería es navegable, con thumbnails, metadata, y búsqueda funcional
